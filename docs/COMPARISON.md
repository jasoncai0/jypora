# jypora vs. Mainstream Markdown Editors

Survey of leading Markdown editors, where jypora stands, and a prioritized
improvement backlog. Editors surveyed: **Typora**(付费, 闭源), **Obsidian**
(免费个人, 闭源, 插件生态), **MarkText**(开源, 停滞), **iA Writer**(付费,
极简写作), **Zettlr**(开源, 学术), **Bear**(付费, Apple 生态), **VS Code +
Markdown 插件**(开源, 开发者向)。

## jypora 当前差距(按用户价值排序)

### P0 — 日常编辑的硬缺口
1. **自动保存未生效**:设置项存在但计时器未接线。所有主流编辑器默认可用。
2. **Copy as Markdown / Copy as HTML**:分享内容的高频操作,缺失。
3. **最近文件**:只有最近工作区;单文件打开无历史,重开成本高。

### P1 — 补齐 Typora 级体验
4. **跨文件全文搜索**:侧边栏搜索目前只匹配文件名;内容级搜索是知识库场景刚需。
5. **图片粘贴落盘**:粘贴图片应保存为 `./assets/<name>.png` 并写相对路径,而非内联 base64(文件膨胀、diff 不可读)。
6. **Find 面板增强**:当前只有计数 + 全部替换;缺高亮定位、上一个/下一个跳转。
7. **未保存关闭确认**:关窗时脏文档应弹确认(数据安全)。

### P2 — 增强与差异化
8. **多标签编辑**:架构改造较大(单文档状态 → 文档集合)。
9. **导出样式定制**:导出 PDF/HTML 跟随当前主题、可选 CSS。
10. **拼写检查开关**、**打字机模式滚动精调**、**footnote 专属 UI**。
11. **插件机制**:主题已可插拔,后续可扩展到渲染器/命令层。
